Each route file exports an async function following Fastify's plugin pattern: `export async function routeName(fastify, opts)` where `opts.db` is the DatabaseSync instance.

**`src/server/routes/projects.js` — GET /api/projects:**
```javascript
export async function projectsRoute(fastify, opts) {
  const { db } = opts;
  fastify.get('/api/projects', async (_request, _reply) => {
    const rows = db.prepare(`
      SELECT id, project_path, last_import_at
      FROM projects
      ORDER BY project_path
    `).all();
    return rows.map(row => ({
      projectId: row.id,
      projectPath: row.project_path,
      displayName: row.project_path.split('/').pop(),
      lastImportAt: row.last_import_at,
    }));
  });
}
```

**`src/server/routes/import.js` — POST /api/import:**

**`src/server/routes/timeline.js` — GET /api/timeline?date=YYYY-MM-DD:**

This is the most complex route. It must:

1. Extract `date` from query string. If missing, use today (`new Date().toISOString().slice(0, 10)`). Validate format is YYYY-MM-DD — if invalid, `reply.code(400); return { error: 'Invalid date format. Use YYYY-MM-DD.' };`

2. Query sessions that overlap the requested date:
```sql
SELECT
  s.session_id,
  s.primary_ticket,
  s.working_branch,
  s.first_message_at,
  s.last_message_at,
  s.message_count,
  s.user_message_count,
  s.fork_count,
  s.real_fork_count,
  p.project_path,
  p.id AS project_id
FROM sessions s
JOIN projects p ON s.project_id = p.id
WHERE DATE(s.first_message_at) <= ? AND DATE(s.last_message_at) >= ?
ORDER BY s.first_message_at
```
Both `?` parameters receive the same `date` value.

3. For each session, fetch message timestamps:
```sql
SELECT timestamp
FROM messages
WHERE session_id = ?
  AND type IN ('user', 'assistant')
  AND timestamp IS NOT NULL
ORDER BY timestamp
```

4. Compute working time using idle-gap algorithm (port from Python `query.py`):
```javascript
const IDLE_THRESHOLD_MS = 10 * 60 * 1000; // 10 minutes
function computeWorkingTime(timestamps) {
  if (timestamps.length < 2) return 0;
  const parsed = timestamps.map(t => new Date(t).getTime());
  let workingMs = 0;
  for (let i = 1; i < parsed.length; i++) {
    const gap = parsed[i] - parsed[i - 1];
    if (gap <= IDLE_THRESHOLD_MS) workingMs += gap;
  }
  return workingMs;
}
```
Define `computeWorkingTime` as a module-level function inside timeline.js (NOT exported — internal to the route).

5. Group sessions by project. Response shape:
```json
{
  "date": "2026-02-26",
  "projects": [
    {
      "projectId": 1,
      "projectPath": "/home/user/myapp",
      "displayName": "myapp",
      "sessions": [
        {
          "sessionId": "abc123",
          "startTime": "2026-02-26T09:00:00.000Z",
          "endTime": "2026-02-26T11:30:00.000Z",
          "workingTimeMs": 4200000,
          "ticket": "AILASUP-42",
          "branch": "meckert-AILASUP-42",
          "messageCount": 87,
          "userMessageCount": 23,
          "forkCount": 2,
          "realForkCount": 1
        }
      ]
    }
  ]
}
```

To build this response: create a `Map<projectId, projectObject>`, iterate sessions, push each to its project's sessions array, then return `{ date, projects: [...map.values()] }`.

**Critical notes:**
- `node:sqlite` DatabaseSync is synchronous — do NOT `await` db calls. The Fastify route handler is async (returns a Promise to Fastify), but database operations inside it are synchronous.
- Prepare statements inside the route handler (not at module level) — the `db` reference comes from `opts` at registration time. Cache the prepared statements in a closure variable at registration time (inside the `timelineRoute` function body but outside the route handler) for efficiency.
