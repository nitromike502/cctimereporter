  <action>
In `src/client/components/DaySummary.vue`, update the `.col-right` CSS rule in the `<style scoped>` section.

Change from:
```css
.col-right {
  text-align: right;
}
```

To:
```css
.col-right {
  text-align: right;
  white-space: nowrap;
  width: 1%;
}
```

The `width: 1%` combined with `white-space: nowrap` causes the column to shrink to exactly fit its content width, giving the first column (name) all remaining space. This is a standard technique for auto-sizing numeric table columns.

Also add `table-layout: auto` explicitly to `.summary-table` for clarity (this is the browser default but makes intent explicit):

The `.summary-table` rule should remain:
```css
.summary-table {
  width: 100%;
  border-collapse: collapse;
  font-size: var(--font-size-sm);
}
```

No change needed — `table-layout: auto` is the default and `width: 100%` is already set.

  <action>
In `src/client/components/DaySummary.vue`, make the following changes:

**1. Update `allSessions` computed to tag sessions with project display name.**

Change from:
```js
const allSessions = computed(() =>
  props.projects.flatMap(p => p.sessions)
)
```

To:
```js
const allSessions = computed(() =>
  props.projects.flatMap(p =>
    p.sessions.map(s => ({ ...s, projectDisplayName: p.displayName }))
  )
)
```

**2. Update `ticketRows` computed to include project names.**

In the `ticketRows` computed, modify the row object construction inside the `for` loop. Change from:
```js
const row = {
  ticket,
  sessionCount: sessions.length,
  workingTimeMs: sessions.reduce((sum, s) => sum + (s.workingTimeMs ?? 0), 0),
}
```

To:
```js
const row = {
  ticket,
  sessionCount: sessions.length,
  workingTimeMs: sessions.reduce((sum, s) => sum + (s.workingTimeMs ?? 0), 0),
  projects: [...new Set(sessions.map(s => s.projectDisplayName))].sort().join(', '),
}
```

**3. Update `branchRows` computed identically.**

In the `branchRows` computed, modify the row object construction. Change from:

```js
const row = {
```
