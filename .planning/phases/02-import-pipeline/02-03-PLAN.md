Create `src/importer/discovery.js` with two exported functions:

**`discoverProjects()`** — Returns array of project objects
- Source 1: Read `~/.claude.json`, parse JSON, get `projects` property (object whose keys are project paths)
  - For each project path, encode it: replace all `/` with `-` to get the transcript directory name
  - Check if `~/.claude/projects/{encoded}` exists as a directory
  - If yes, add to result with `{ projectPath, transcriptDir: fullPath }`
  - Wrap the ~/.claude.json read in try/catch — if file missing or malformed, log warning and continue with Source 2 only
- Source 2: Scan `~/.claude/projects/` filesystem for ALL directories
  - For any directory NOT already found via Source 1, add as orphaned project
  - For orphaned projects: use the directory name as projectPath (do NOT try to decode — the encoding is lossy per RESEARCH.md)
  - `{ projectPath: dirName, transcriptDir: fullPath }`
- Deduplicate by transcript directory path
- Return the merged array sorted by projectPath

**`findTranscriptFiles(transcriptDir)`** — Returns array of file objects for a single project
- Read directory contents with `readdirSync(transcriptDir)`
- Filter: must end with `.jsonl`, must NOT start with `agent-`, must be a file (not directory — use `statSync().isFile()`)
- For each matching file, return:
  ```javascript
  {
    name: filename,
    path: fullPath,
    sessionId: filename.slice(0, -6),  // strip .jsonl extension
    size: statSync(fullPath).size,
  }
  ```
- Wrap the entire function in try/catch — if directory is unreadable, log warning and return empty array

**Important edge cases (from RESEARCH.md):**
- `sessions-index.json` in transcript directories — filtered out by .jsonl extension check
- Session subdirectories (UUID without extension) — filtered out by isFile() check
- `agent-*` prefixed files in some layouts — filtered out by startsWith check

<task type="auto">
  <name>Task 2: Import orchestrator wiring the full pipeline</name>
  <files>src/importer/index.js</files>
  <action>
Create `src/importer/index.js` that exports `importAll(db, options)` — the main import entry point.

**Parameters:**
- `db`: DatabaseSync instance (already opened with v2 schema)
- `options`: `{ force: boolean, verbose: boolean }` — force re-imports all files ignoring size check; verbose prints progress

**Algorithm:**

1. **Discover projects:**
   ```javascript
   const projects = discoverProjects();
   ```

2. **For each project:**
   a. Get or create project record in `projects` table:
      - Query: `SELECT id FROM projects WHERE project_path = ?`
      - If not found: `INSERT INTO projects (project_path, transcript_dir) VALUES (?, ?)`
      - Get the project_id
   b. Find transcript files: `findTranscriptFiles(project.transcriptDir)`

3. **Filter files for import (size-based skip):**
   - Call `getImportedFileSizes(db)` to get the Map of previously imported file paths and sizes
   - If `!options.force`: skip files where `importedSizes.get(file.path) === file.size`
   - If `options.force`: process all files

4. **For each file to import:**
   a. Parse: `const data = await parseTranscript(file.path)`
   b. Compute derived data:
      - `timedMessages`: messages with non-null timestamp
      - `userMessages`: timedMessages where type === 'user' && !isMeta
      - `assistantMessages`: timedMessages where type === 'assistant'
      - `workingBranch`: call `determineWorkingBranch(data.messages)`
      - `primaryTicket`: call `scoreTickets(data.messages, workingBranch)`
      - `forkData`: call `detectForks(data.messages)`
      - `firstMessageAt` / `lastMessageAt`: min/max of timestamps from timedMessages (sort as strings — ISO8601 sorts lexicographically)
      - `toolUseCount`: count assistant messages whose rawMessage.message.content array has items with type === 'tool_use'
   c. Upsert session: call `upsertSession(db, { session_id, project_id, file_path, file_size, working_branch, primary_ticket, summary, custom_title, slug, first_message_at, last_message_at, message_count, user_message_count, assistant_message_count, tool_use_count, fork_count, real_fork_count, is_compacted, has_subagents })`
   d. Insert messages: call `insertMessages(db, sessionId, messagesForDb)` where each message is mapped to the db-writer format:
      ```javascript
      {
        uuid: msg.uuid,
        type: msg.type,
        subtype: msg.subtype,
        timestamp: msg.timestamp,
        parent_uuid: msg.parentUuid,
        git_branch: msg.gitBranch,
        is_meta: msg.isMeta ? 1 : 0,
        is_sidechain: msg.isSidechain ? 1 : 0,
        is_fork_branch: forkData.forkBranchUuids.has(msg.uuid) ? 1 : 0,
      }
      ```
   e. Collect and upsert tickets: scan messages for ticket detections, build unique (ticket_key, source, detected_at) tuples, call `upsertTickets(db, sessionId, tickets, primaryTicket)`
   f. Update import log: call `updateImportLog(db, file.path, sessionId, file.size, 'ok', null)`
   g. If any step fails for a file: catch error, call `updateImportLog(db, file.path, sessionId, file.size, 'error', error.message)`, log warning, continue with next file

5. **Progress reporting (if verbose):**
   - Before each file: `process.stderr.write('  Importing ${sessionId.slice(0,8)}...\n')`
   - After each file: `process.stderr.write('  ${messageCount} messages, branch: ${workingBranch || 'none'}, ticket: ${primaryTicket || 'none'}\n')`
   - Final summary: `process.stderr.write('Import complete: ${filesProcessed} files, ${totalMessages} messages\n')`

6. **Return summary object:**
   ```javascript
   { projectsFound, filesProcessed, filesSkipped, totalMessages, errors: [...] }
   ```

**Ticket collection for upsertTickets:**
Port the Python PoC's ticket detection loop. For each message, check:
- User messages: scan content for /prep-ticket (inline and XML), scan for generic TICKET_PATTERN mentions
- All messages with gitBranch: scan gitBranch for TICKET_PATTERN
- Build a Set of `${ticket_key}|${source}` to deduplicate, then convert to array of {ticket_key, source, detected_at}

Use the TICKET_PATTERN and prep-ticket patterns from ticket-scorer.js (import them).

**Update project last_import_at:**
After processing all files for a project, update `projects SET last_import_at = datetime('now') WHERE id = ?`

**Critical implementation notes:**
- `parseTranscript` is async (readline streaming) — the import loop must use `await`
- `importAll` itself must be async
- All db operations are synchronous (node:sqlite is sync) — no await needed for db calls
- Do NOT wrap the entire import in a single transaction — each file gets its own transaction via insertMessages, and individual upserts are auto-committed
