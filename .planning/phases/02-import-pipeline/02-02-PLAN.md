  <action>
Create `src/importer/parser.js` that exports an async function `parseTranscript(filePath)`.

**Implementation — port of Python PoC `parse_transcript()`:**

Use `node:readline` with `createReadStream` for streaming:
```javascript
import { createReadStream } from 'node:fs';
import { createInterface } from 'node:readline';
```

The function streams a JSONL file line-by-line and returns:
```javascript
{
  messages: [...],      // Array of parsed message objects
  summary: string|null,
  customTitle: string|null,
  slug: string|null,
  hasCompactBoundary: boolean,
  hasSubagents: boolean,
}
```

**For each valid JSONL line, extract a message object:**
```javascript
{
  uuid: msg.uuid || `line-${lineNum}`,  // Fallback for messages without UUID
  type: msg.type,
  subtype: msg.subtype || null,
  timestamp: msg.timestamp || null,
  parentUuid: msg.parentUuid || null,
  gitBranch: msg.gitBranch || null,
  cwd: msg.cwd || null,
  version: msg.version || null,
  isMeta: msg.isMeta || false,
  isSidechain: msg.isSidechain || false,
  isCompactSummary: msg.isCompactSummary || false,
  agentId: msg.agentId || null,
  sourceToolAssistantUuid: msg.sourceToolAssistantUUID || null,  // Note: uppercase UUID in source
  rawMessage: msg,  // Keep full message for content extraction
}
```

**Session-level metadata extraction (during parse):**
- `summary`: from `type === 'summary'` messages, take `msg.summary`
- `customTitle`: from `type === 'custom-title'` messages, take `msg.customTitle`
- `slug`: first message with a `slug` field
- `hasCompactBoundary`: any message with `type === 'system' && subtype === 'compact_boundary'`
- `hasSubagents`: any message with `isSidechain || agentId`

**Error handling:**
- Skip blank lines (continue)
- Wrap `JSON.parse(line)` in try/catch — on error, write warning to stderr: `Warning: malformed JSONL line ${lineNum} in ${filePath}: ${error.message}`, then continue
- Never throw for malformed lines — the import must continue

**Also export a helper `extractContentText(msg)`:**
- Takes a raw message object (the full JSONL line object, NOT the parsed message)
- Extracts text content for ticket scanning:
  - If `msg.message.content` is a string, return it
  - If it's an array, filter for `type === 'text'` blocks, join their `.text` values with newline
  - Otherwise return `JSON.stringify(msg.message.content)`
  - If no content found, return empty string

**fork-detector.js — port of Python PoC `detect_forks()`:**

Export `detectForks(messages)` that takes the messages array from parseTranscript and returns:
```javascript
{ forkCount: number, realForkCount: number, forkBranchUuids: Set<string> }
```

Algorithm (direct port from Python PoC):
1. Build `childrenMap`: Map of parentUuid -> [childUuid, ...] from all messages with both uuid and parentUuid
2. Build `msgByUuid`: Map of uuid -> message object
3. For each entry in childrenMap where childUuids.length >= 2 (a fork point):
   a. Check if it's a progress fork: all non-first children have type 'progress' or 'file_history_snapshot'
   b. Increment forkCount
   c. If NOT a progress fork, increment realForkCount and mark secondary branch descendants:
      - Count descendants for each child branch using iterative DFS (stack-based, not recursive)
      - Sort by descendant count descending — first is primary, rest are secondary
      - Walk all secondary branch descendants and add their UUIDs to forkBranchUuids set
4. Return { forkCount, realForkCount, forkBranchUuids }

**ticket-scorer.js — port of Python PoC `determine_primary_ticket()` with generic pattern:**

Export `scoreTickets(messages, workingBranch)` that returns the primary ticket key (string) or null.

Constants (exported for potential reuse):
```javascript
export const TICKET_PATTERN = /[a-zA-Z]{2,8}-\d+/gi;
const PREP_TICKET_INLINE = /\/prep-ticket\s+([a-zA-Z]{2,8}-\d+)/i;
const PREP_TICKET_XML = /<command-name>\/prep-ticket<\/command-name>.*?<command-args>([a-zA-Z]{2,8}-\d+)<\/command-args>/is;
```

**IMPORTANT: Ticket pattern is `[a-zA-Z]{2,8}-\d+` (CONTEXT.md locked decision), NOT `AILASUP-\d+`.**

Algorithm:
1. Find firstUserMsg: first message where type === 'user' && !isMeta
2. **Branch base scoring:** If workingBranch contains TICKET_PATTERN match, add 100 points for that ticket
3. **Per-message scoring:** Iterate all messages:
   a. If msg.gitBranch matches TICKET_PATTERN: add 5 points per match (branch frequency bonus)
   b. If msg.type === 'user': extract content text using extractContentText from parser.js:
      - Check for /prep-ticket (both inline and XML patterns): add 500 points (700 if msg.uuid === firstUserMsg.uuid)
      - Check for generic content mentions via TICKET_PATTERN: add 10 points per match
4. All ticket keys are uppercased before scoring (AILASUP-123 === ailasup-123)
5. Return the ticket with the highest score, or null if no tickets found

**Import extractContentText from parser.js** — this is the content extraction helper.

Also export a helper `determineWorkingBranch(messages)` that:
1. Counts occurrences of each gitBranch across all messages, skipping: null, 'main', 'master', 'develop', 'dev', 'staging', and branches starting with 'project-' or 'target-version-'
2. Prefers branches containing a TICKET_PATTERN match (return first such branch found by count)
3. Falls back to the most common non-skipped branch
4. Returns null if no valid branches
  </action>
