# Phase 13-01: Parse Slash Command XML in Session Summaries

## Problem
Many sessions start with slash commands that produce XML tags like:
```
<command-name>/gsd:execute-phase</command-name>
<command-message>gsd:execute-phase</command-message>
<command-args>7</command-args>
```
The current `SYNTHETIC_MSG_RE` in parser.js skips these entirely, leaving 19/89 sessions with null `first_prompt`.

## Approach: Dual — Import-time + Display-time

### 1. Import-time: Parse XML in parser.js (primary)
Modify the `firstPrompt` capture logic in `parseTranscript()`:
- Instead of skipping messages matching `SYNTHETIC_MSG_RE`, try to parse XML command tags first
- Extract `<command-name>` → command name, `<command-args>` → arguments
- Produce readable text like `/gsd:execute-phase 7`
- Only skip truly unparseable synthetic messages (teammate-message, local-command)

### 2. Display-time: Utility function for existing data
Add a `parseCommandXml()` utility that can be used client-side to clean up already-imported data without requiring re-import.
Apply it in `SessionDetailPanel.vue` and `GanttBar.vue` summaryText computeds.

## Real Examples Found
```
"<command-name>/clear</command-name>\n<command-message>clear</command-message>\n<command-args></command-args>"
"<command-message>dev-workflow</command-message>\n<command-name>/dev-workflow</command-name>\n<command-args>story-2.4</command-args>"
"<command-message>gsd:new-project</command-message>\n<command-name>/gsd:new-project</command-name>"
```

## Implementation

### File: `src/utils/parse-command-xml.js` (new shared utility)
- Export `parseCommandXml(text)` — returns cleaned string or null if not command XML
- Regex extracts `<command-name>` and `<command-args>` tags
- Returns `/command-name args` format

### File: `src/importer/parser.js`
- Import `parseCommandXml`
- In firstPrompt capture: try parsing XML before applying SYNTHETIC_MSG_RE skip
- Narrow SYNTHETIC_MSG_RE to only `teammate-message` and `local-command`

### File: `src/client/components/SessionDetailPanel.vue`
- Import client-side copy of parseCommandXml
- Apply in summaryText computed for display-time cleanup

### File: `src/client/components/GanttBar.vue`
- Same display-time parsing for the session label

## Verification
- `npm run build` passes
- Re-import populates previously null first_prompt values
