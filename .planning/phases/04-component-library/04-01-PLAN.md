**Update `src/server/index.js`** to serve the built Vue app via `@fastify/static`:

1. Add imports at the top:
```js
import fastifyStatic from '@fastify/static';
import { fileURLToPath } from 'node:url';
import { join, dirname } from 'node:path';
```

2. Compute dist path:
```js
const __dirname = dirname(fileURLToPath(import.meta.url));
const distPath = join(__dirname, '../../dist');
```

3. Inside `createServer(db)`, AFTER registering all API routes (timeline, projects, import), add:
```js
  // Serve built Vue SPA from dist/
  app.register(fastifyStatic, {
    root: distPath,
    wildcard: false,  // Required: prevents conflict with setNotFoundHandler
  });

  // SPA catch-all: any non-API, non-file request serves index.html
  app.setNotFoundHandler((_req, reply) => {
    reply.sendFile('index.html');
  });
```

**Critical ordering:** API routes MUST be registered BEFORE `@fastify/static`. The existing code already registers API routes first — the static plugin goes after all three route registrations.

**Critical setting:** `wildcard: false` is mandatory. Without it, `@fastify/static` registers its own wildcard handler that conflicts with `setNotFoundHandler`. The SPA catch-all needs `setNotFoundHandler` to serve `index.html` for client-side routes like `/components` and `/timeline`.

**Do NOT change anything else in server/index.js** — the existing API route registrations and Fastify config stay the same.
