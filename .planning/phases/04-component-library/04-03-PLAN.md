**Create `src/client/components/AppDatePicker.vue`:**

Props:
- `modelValue`: [Date, String, null] — the selected date
- `disabled`: Boolean (default: false)
- `placeholder`: String (default: `'Select date'`)

Emits: `update:modelValue`

Uses `@vuepic/vue-datepicker`:
```js
import VueDatePicker from '@vuepic/vue-datepicker'
import '@vuepic/vue-datepicker/dist/main.css'
```

Template: Wrap `<VueDatePicker>` with these props:
- `v-model="model"` (using `defineModel()`)
- `:disabled="disabled"`
- `:placeholder="placeholder"`
- `auto-apply` (selects date on click, no confirmation button)
- `:enable-time-picker="false"` (date only, no time selection)
- `:dark="isDark"` — detect dark mode via `window.matchMedia('(prefers-color-scheme: dark)')` and pass as a reactive ref. Listen for changes with `matchMedia.addEventListener('change', ...)` to react to OS theme switches.

**CRITICAL: CSS overrides must be in a NON-SCOPED `<style>` block.** Scoped CSS cannot penetrate `@vuepic/vue-datepicker` internal DOM. Use a separate `<style>` (without `scoped`) for the vendor overrides.

CSS overrides for the light theme (`.dp__theme_light`):
```css
.dp__theme_light {
  --dp-primary-color:      var(--color-primary);
  --dp-primary-text-color: var(--color-navy);
  --dp-hover-color:        color-mix(in srgb, var(--color-primary) 25%, transparent);
  --dp-hover-text-color:   var(--color-heading);
  --dp-border-color:       var(--color-border);
  --dp-background-color:   var(--color-bg);
  --dp-text-color:         var(--color-body-text);
  --dp-border-radius:      var(--radius-md);
  --dp-font-family:        var(--font-family);
  --dp-font-size:          var(--font-size-sm);
}
```

CSS overrides for the dark theme (`.dp__theme_dark`):
```css
.dp__theme_dark {
  --dp-primary-color:      var(--color-primary);
  --dp-primary-text-color: var(--color-navy);
  --dp-hover-color:        color-mix(in srgb, var(--color-primary) 25%, transparent);
  --dp-hover-text-color:   var(--color-heading);
  --dp-border-color:       var(--color-border);
  --dp-background-color:   var(--color-bg);
  --dp-text-color:         var(--color-body-text);
  --dp-border-radius:      var(--radius-md);
  --dp-font-family:        var(--font-family);
  --dp-font-size:          var(--font-size-sm);
}
```

Both themes override the same vars — the design tokens themselves already switch for dark mode via `prefers-color-scheme` in tokens.css. The `dark` prop on `VueDatePicker` selects which `.dp__theme_*` class is applied to the calendar popup.

Optional scoped `<style scoped>` for the wrapper div (if any outer styling is needed, like width constraints).
