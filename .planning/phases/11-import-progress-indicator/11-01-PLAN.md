Add a GET route inside the existing `importRoute` plugin function (same file, same `importRunning` guard):

1. Register `fastify.get('/api/import/progress', async (request, reply) => { ... })` inside the existing `importRoute` function, AFTER the POST route.

2. Implementation:
   - Check `importRunning` — if true, return 409 JSON `{ error: 'Import already in progress' }`.
   - Set `importRunning = true`.
   - Call `reply.hijack()` to take over the response.
   - Write SSE headers to `reply.raw`: Content-Type `text/event-stream`, Cache-Control `no-cache`, Connection `keep-alive`.
   - Track client disconnect: `let clientConnected = true; request.raw.on('close', () => { clientConnected = false; });`
   - Create `sendEvent(eventName, data)` helper: if clientConnected, write `event: ${eventName}\ndata: ${JSON.stringify(data)}\n\n` to `reply.raw`.
   - Parse `maxAgeDays` from `request.query` (parseInt, pass undefined if absent).
   - Call `importAll(db, { maxAgeDays, onProgress(progress) { sendEvent('progress', progress); } })`.
   - On success: `sendEvent('complete', result)`.
   - On error: `sendEvent('error', { message: err.message })`.
   - In finally block: set `importRunning = false`, call `reply.raw.end()`.

3. Keep the existing POST /api/import route unchanged — it remains as a non-streaming fallback.
