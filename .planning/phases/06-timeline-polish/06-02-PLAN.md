  <name>Task 1: Clip overnight sessions to day boundaries in timeline API</name>
  <files>src/server/routes/timeline.js</files>
  <action>
    In the `timelineRoute` handler, after building `sessionObj` and before pushing to projectMap:

    1. Compute day boundaries:
       ```js
       const dayStartISO = date + 'T00:00:00';
       const dayEndISO = date + 'T23:59:59.999';
       ```

    2. Clamp startTime and endTime to the day:
       ```js
       const clampedStart = sessionObj.startTime < dayStartISO ? dayStartISO : sessionObj.startTime;
       const clampedEnd = sessionObj.endTime > dayEndISO ? dayEndISO : sessionObj.endTime;
       sessionObj.startTime = clampedStart;
       sessionObj.endTime = clampedEnd;
       ```

    3. Filter idle gaps to only include gaps that overlap with the visible day:
       ```js
       sessionObj.idleGaps = sessionObj.idleGaps
         .map(g => ({
           start: g.start < dayStartISO ? dayStartISO : g.start,
           end: g.end > dayEndISO ? dayEndISO : g.end,
         }))
         .filter(g => g.start < g.end);
       ```

    4. Recompute workingTimeMs for the clamped range: filter the `timestamps` array to only include those within `[dayStartISO, dayEndISO]` BEFORE calling `computeWorkingTime`. Move the timestamps filtering before the workingTimeMs computation:
       ```js
       const clampedTimestamps = timestamps.filter(t => t >= dayStartISO && t <= dayEndISO);
       const workingTimeMs = computeWorkingTime(clampedTimestamps);
       ```
       Then use `clampedTimestamps` for both `computeWorkingTime` and `computeIdleGaps`.

    This ensures the API always returns session data scoped to the requested date. No client changes needed — GanttBar's timeToPercent already clamps to 0-100%.
  </action>
  <verify>
    Start the server, call `curl http://localhost:3847/api/timeline?date=YYYY-MM-DD` for a date with an overnight session. Verify the response has startTime/endTime within that day and idleGaps are clipped.
  </verify>
  <done>
    Overnight sessions return startTime >= dayStart and endTime <= dayEnd. Idle gaps outside the day are removed. Working time is computed only for the visible day's messages.
  </done>
