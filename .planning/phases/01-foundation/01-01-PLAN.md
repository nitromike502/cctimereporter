    - Call `openDatabase()` to ensure database is created
    - Print `cctimereporter v0.1.0` to stdout
    - Exit with code 0

    IMPORTANT: Do NOT use static `import` for node:sqlite-dependent modules in bin/cli.js — ESM hoists them before any code runs, defeating the version check. Use `await import()` (top-level await is supported in ESM).

<task type="auto">
  <name>Task 2: Database layer with minimal v1 schema</name>
  <files>
    src/db/schema.js
    src/db/index.js
  </files>
  <action>
    Create `src/db/schema.js`:
    - Export `SCHEMA_VERSION = 1` (integer constant)
    - Export `SCHEMA_DDL` as a template literal string containing CREATE TABLE IF NOT EXISTS for these 3 tables:

    **projects table:**
    - `id INTEGER PRIMARY KEY AUTOINCREMENT`
    - `project_path TEXT NOT NULL UNIQUE` — original path like /home/user/project
    - `transcript_dir TEXT NOT NULL` — encoded dir name like -home-user-project
    - `last_import_at TEXT` — nullable, ISO 8601
    - `created_at TEXT NOT NULL DEFAULT (datetime('now'))`
    - Index on project_path

    **sessions table:**
    - `id INTEGER PRIMARY KEY AUTOINCREMENT`
    - `session_id TEXT NOT NULL UNIQUE` — UUID from filename
    - `project_id INTEGER NOT NULL` — FK to projects
    - `file_path TEXT NOT NULL` — full path to .jsonl file
    - `file_modified_at TEXT` — for change detection
    - `working_branch TEXT`
    - `primary_ticket TEXT`
    - `summary TEXT`
    - `first_message_at TEXT`
    - `last_message_at TEXT`
    - `message_count INTEGER DEFAULT 0`
    - `user_message_count INTEGER DEFAULT 0`
    - `tool_use_count INTEGER DEFAULT 0`
    - `imported_at TEXT NOT NULL DEFAULT (datetime('now'))`
    - `FOREIGN KEY (project_id) REFERENCES projects(id)`
    - Indexes on: first_message_at, project_id, primary_ticket

    **messages table:**
    - `id INTEGER PRIMARY KEY AUTOINCREMENT`
    - `uuid TEXT NOT NULL`
    - `session_id TEXT NOT NULL` — FK to sessions.session_id
    - `type TEXT NOT NULL` — user, assistant, etc.
    - `timestamp TEXT NOT NULL` — ISO 8601
    - `git_branch TEXT`
    - `is_sidechain BOOLEAN DEFAULT 0`
    - `UNIQUE(session_id, uuid)`
    - `FOREIGN KEY (session_id) REFERENCES sessions(session_id)`
    - Indexes on: session_id, timestamp

    This is the MINIMAL schema for v1. Do NOT include fork_points, tool_uses, tickets, import_log, views, or schema_version table from the PoC. Those are explicitly deferred per CONTEXT.md decisions.

    Create `src/db/index.js`:
    - Import `DatabaseSync` from `node:sqlite`
    - Import `mkdirSync`, `unlinkSync`, `existsSync` from `node:fs`
    - Import `join` from `node:path`, `homedir` from `node:os`
    - Import `SCHEMA_DDL`, `SCHEMA_VERSION` from `./schema.js`
    - Define `DB_DIR = join(homedir(), '.cctimereporter')`
    - Define `DB_PATH = join(DB_DIR, 'data.db')`
    - Export `openDatabase()` function that:
      1. Creates DB_DIR with `mkdirSync(DB_DIR, { recursive: true })`
      2. Opens database with `new DatabaseSync(DB_PATH)`
      3. Checks `PRAGMA user_version` — if it does not match SCHEMA_VERSION, close db, delete file, reopen
      4. Wraps the open in try/catch — on any error (corruption), delete and recreate
      5. Sets `PRAGMA journal_mode = WAL`
      6. Sets `PRAGMA foreign_keys = ON`
      7. Executes `SCHEMA_DDL` via `db.exec()`
      8. Sets `PRAGMA user_version = ${SCHEMA_VERSION}` via `db.exec()` (NOT prepared statement — PRAGMA cannot use parameter binding)
      9. Returns the DatabaseSync instance
    - Also export `DB_PATH` for use by other modules (testing, logging)

    IMPORTANT: Use `db.exec()` not `db.prepare().run()` for PRAGMA user_version SET — parameter binding does not work with PRAGMA statements.
  </action>
  <verify>
    Run `node bin/cli.js` — database file should be created at ~/.cctimereporter/data.db.
    Run `node -e "import { openDatabase } from './src/db/index.js'; const db = openDatabase(); const tables = db.prepare(\"SELECT name FROM sqlite_master WHERE type='table' ORDER BY name\").all(); console.log(JSON.stringify(tables));"` — should list messages, projects, sessions tables.
    Delete ~/.cctimereporter/data.db and run `node bin/cli.js` again — should recreate cleanly.
  </verify>
  <done>
    - Database file created at ~/.cctimereporter/data.db on first run
    - Database contains exactly 3 tables: messages, projects, sessions
    - WAL mode is active (PRAGMA journal_mode returns 'wal')
    - Foreign keys are enabled (PRAGMA foreign_keys returns 1)
    - Deleting and re-running recreates the database without errors
    - Schema version stored via PRAGMA user_version = 1
  </done>
</task>
