**tokens.css:** Replace the `@media (prefers-color-scheme: dark) { :root { ... } }` block (lines 50-62) with a `[data-theme="dark"] { ... }` block. Keep the exact same dark token values. The `:root` light tokens block stays as-is. Do NOT keep a @media fallback — the inline script handles system preference detection.

**index.html:** Add an inline `<script>` tag inside `<head>`, BEFORE the closing `</head>` tag. The script must:
1. Read `localStorage.getItem('cctimereporter:theme')`
2. If no saved value, check `window.matchMedia('(prefers-color-scheme: dark)').matches`
3. Set `document.documentElement.setAttribute('data-theme', theme)` where theme is 'dark' or 'light'
4. Use `var` not `const`/`let` (broadest compat for inline scripts)
5. Use an IIFE to avoid global scope pollution

**useTheme.js:** Create `src/client/composables/useTheme.js` (create the `composables/` directory). Implementation:
- Module-level `const isDark = ref(document.documentElement.getAttribute('data-theme') === 'dark')`
- Module-level `watch(isDark, ...)` that sets `data-theme` attribute and writes to `localStorage` key `cctimereporter:theme`
- Exported `useTheme()` function returns `{ isDark, toggle }` where `toggle` flips `isDark.value`
- The watcher and ref are singletons (module-level) — multiple `useTheme()` calls share the same state and only one watcher runs

Import `ref` and `watch` from `vue`.
