# Phase 14-01: Session Message Modal

## Goal
Make the session summary clickable in SessionDetailPanel. When clicked, open a modal showing the first messages of the session (up to 10 messages or first tool execution, whichever comes first).

## Key Discovery
The `messages` table stores only metadata (uuid, type, subtype, timestamp) — **no content**. To show message text, the API must read the original JSONL file referenced by `sessions.file_path`.

## Implementation

### A. New API Endpoint: `GET /api/sessions/:id/messages`

**File:** `src/server/routes/messages.js`

1. Query `sessions` table for `file_path` by `session_id`
2. Stream the JSONL file using `node:readline` (same pattern as parser.js)
3. For each line, collect user/assistant messages:
   - Extract text content from `message.content` (string or array of text blocks)
   - Skip: progress, system, file-history-snapshot, summary, queue-operation, custom-title
   - Skip: messages with `isMeta`, `isSidechain`, `isCompactSummary`
   - **Stop condition:** 10 collected messages OR first assistant message containing a `tool_use` block
4. Return `{ messages: [{ role, content, timestamp }] }`
5. Register in `src/server/index.js`

### B. Modal Component: `SessionMessagesModal.vue`

**File:** `src/client/components/SessionMessagesModal.vue`

Uses Reka UI Dialog primitives (DialogRoot, DialogPortal, DialogOverlay, DialogContent, DialogTitle, DialogClose).

- Props: `open` (boolean), `sessionId` (string)
- Emits: `update:open`
- When open + sessionId changes, fetch `/api/sessions/{id}/messages`
- Display messages in scrollable list with User/Assistant labels
- Loading and error states
- Style with existing design tokens

### C. Integration in SessionDetailPanel.vue

- Make summary text clickable (cursor pointer, subtle hover underline)
- On click, emit event to parent
- In TimelinePage.vue, add SessionMessagesModal and wire up open/close state
